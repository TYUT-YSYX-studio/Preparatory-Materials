> ### 动态数组
>
> #### malloc/free
>
> 在大家学习指针的时候，有接触到内存的概念，那么C语言有没有什么函数来帮助我们操作内存呢，那就是`malloc`和`free`。
>
> 而操控内存到底有什么用处呢，想象一下：你要办一场聚会，提前在家摆好固定数量的椅子
>
> - 你猜可能来 10 个人，就摆了 10 把椅子。
>
> - 结果来了 20 个人 → 没椅子坐，客人站门口，很尴尬！
>
> - 或者只来了 3 个人 → 多出 7 把空椅子，浪费空间。
>
> 这就像写 C 程序时提前定义一个固定大小的数组：
>
> ```C
> int guests[10]; *// 最多只能存10个人的名字*
> ```
>
> 但如果你不知道到底会来多少人，这种“死板”的方式就不够用了，这时候就是malloc派上用场了
>
>
>
> `malloc`和`free`都在`stdlib.h`中被声明，使用前需引用该头文件。
>
> `malloc`就是用来分配内存的函数，它的功能是让程序在运行的时候，自己决定要多少内存，而不是一开始就定死，它的函数声明如下：
>
> ```C
> void *malloc(size_t size);
> ```
>
> 可见`malloc`是个指针函数，其返回的是指向`从分配的内存起始位置`的指针，不过是`void*`类型的指针。其往后的`size`大小**字节**的内存是已分配且未初始化的。
>
> `void*`类型表示其指向的那段内存将可以当做任何类型的变量进行处理。
>
> `malloc`返回的是`void *`*，不过为了更安全地使用malloc返回的内存指针，我们还最好还是进行一个强制转换，将**`void *`*转换为`int *`，我们给出如下代码：
>
> ```C
> int *guests = NULL;
> guests = (int*)malloc(10 * sizeof(int));//申请10个整数类型的内存大小
> ```
>
>
>
> 这里为guests指针赋值了内存的地址，让其指向一段已分配的内存。`(int*)` 是告诉程序：“我打算把这块内存当成整数来用”，这时候就可以把guests当做数组首元素指针使用了，只是这个数组只有10个成员。不过由于内存中的数据没被初始化，这时候访问到数组内的元素谁也不知道里面会是什么（虽然有不小概率会是0）。
>
> 这里使用的NULL其实是一个空指针，但C语言和C\+\+的空指针概念有一定的区别，如果你还辅修了C\+\+一定不要搞混了。
>
> C语言中的NULL空指针指向地址为0的那段内存（一般不可访问，会导致segmentation fault），也就是指针变量本身的值是0。这和我们平时顺手初始化一个int类型变量为0时没有什么不同。
>
> ```C
> int *guests = NULL;
> guests = (int*)malloc(10 * sizeof(int));//申请10个整数类型的内存大小
> if (guests == NULL) {// 处理分配失败的情况
>     printf("内存分配失败！\n");
>     return -1;
> }
> ```
>
> malloc可能返回NULL（当系统内存不足时），所有以每次分配完都应该检查分配是否成功。
>
> malloc分配的内存是不会自动回收的！既然要手动分配，那自然也要手动释放，这时候就需要用到free函数了。修改后的代码如下：
>
> ```C
> int *guests = NULL;
> guests = (int*)malloc(10*sizeof(int));
> if (guests == NULL) {// 处理分配失败的情况
>     printf("内存分配失败！\n");
>     return -1;
> }
> ......
> free(guests);
> guests = NULL;
> ```
>
> 在使用完这块内存并保证不再使用它时，我们就要用free去释放这块内存，并将guests设为NULL防止后续代码继续访问这块内存。

> [!TIP]
> # 为什么需要free？
>
> 前面提到过，malloc返回的指针是指向一段内存的起始。但是为什么我们还需要用free进行释放呢？难道这些内存还被标记了？的确如此，已分配内存的会被标识该区域是否正被使用。
>
> 假如没有free，那么这段内存将始终处于被使用的状态，长此以往可能就会导致程序再无内存可用，直到程序崩溃。这也就是常说的内存泄漏带来的严重后果。

> ##### 结构体指针
>
> 结构体变量当然也是变量，也有自己的内存起始地址。但我们更需要一个指针指示这段内存储存的是一个什么样的结构体。
>
> 结构体指针的定义规则如下：
>
> ```C
> struct struct_name *pointer = struct_address;                                
> ```
>
> 具体地，可以有这样的结构体指针：
>
> ```C
> struct student {
>     const char *name;
>     unsigned int  grade;
> } S = {};
>
> struct student *p = &S;
> ```
>
> 要访问结构体指针指向的结构体的成员，需要用\-\>符号，用例如下：
>
> ```C
> struct student {
>     const char *name;
>     unsigned int  grade;
> } S = {};
>
> struct student *p = &S;
>
> int main(){
>     p->name = NULL;
>     p->grade = 0;
> }
> ```
>
> 别忘记了，指针也是变量，其本质也只是一段数据，只是这次指向的变量更大更特殊，仅此而已。
>
> 以及，不要忘记结构体数组有关知识，并结合前面的动态内存分配完成大作业。
>
> PS：尽管你可以使用一个非常大的数组来完成大作业，但这并不优雅。

> 老师身份可以向所有学生发布问卷，学生那里可以回答问卷，回答完毕后老师身份能得到问卷及回答情况
>
> 1. 老师身份编辑问卷的问题（限制字数）
>
> 2. 学生视角登录该模块，可以看到未回答的问卷，选择指定问卷进行回答
>
> 3. 回答后学生视角消失，不能重复作答
>
> 4. 回答后，老师模块可以查看公布的问卷，得到学生回答情况，随后自动删除
>
>
>
> #### 退出模块
>
> 无论老师和学生身份，在上面事务进行完之后，可以安全退出管理系统，提醒安全退出后结束程序运行
>
> 以上所有功能能实现多少实现多少，难度是肯定有的，交作业时用代码功能txt文档说清楚实现了哪些功能即可
>
> 同时，请注意这是用户程序，一个不读你原码的人应该只通过你的输出信息，就能知道该程序应该怎么使用，逼人读开发手册可不是一个好程序，因此**合理的程序运行提醒**很重要
